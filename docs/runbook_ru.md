## Смоук-тест в тестнете Nile

Цель: доказать, что весь поток работает сквозным образом до выхода в мейннет.

### Предварительные требования

- [ ] Поднята БД тестнета Nile (Neon или локальный postgres)
- [ ] TRON-аккаунты тестнета Nile:
  - Сгенерирован депозитный xprv (тестовые монеты не имеют реальной ценности, но используйте свежий ключ)
  - Приватный ключ горячего кошелька (пополнен ≥100 Nile TRX + 100 Nile USDT)
  - Адрес холодного кошелька (только адрес, ключ в env не нужен)
- [ ] API-ключ TronGrid (бесплатного тарифа для Nile достаточно)
- [ ] Поднят Upstash Redis KV (бесплатного тарифа достаточно)
- [ ] Токен Telegram-бота от @BotFather (используйте тестового бота, не продакшен)
- [ ] ngrok или похожий туннель для доставки webhook
- [ ] `.env.local` заполнен всеми обязательными переменными, плюс:
  - `DEPLOY_URL=<ngrok https URL>`
  - `CRON_SECRET` и `ADMIN_API_SECRET` (сгенерируйте через `openssl rand -hex 32`)
  - `TELEGRAM_WEBHOOK_SECRET` (сгенерируйте через `openssl rand -hex 16`)

### Шаг 1: Предполётная проверка

```bash
# Проверить, что окружение заполнено
npx tsx scripts/e2e-nile.ts
```

Все проверки должны пройти. Исправьте всё, что отмечено, прежде чем продолжать.

### Шаг 2: Миграции + сид

```bash
npx tsx scripts/migrate.mts
npx tsx scripts/seed.ts
```

Проверьте:
```sql
SELECT * FROM subscription_plans;   -- 3 строки
SELECT * FROM commission_config;    -- 1 строка
SELECT * FROM ops_kill_switch;       -- 1 строка (false/false)
```

### Шаг 3: Webhook

```bash
npx tsx scripts/setup-telegram-webhook.ts <ngrok https URL>

# Проверка
curl https://api.telegram.org/bot<TOKEN>/getWebhookInfo
```

Дождитесь `"ok":true` с `"url"`, указывающим на ваш ngrok-URL.

### Шаг 4: Запуск сервера разработки

```bash
npm run dev
```

Оставьте этот терминал открытым. Все запросы маршрутизируются через ngrok → localhost:3000.

### Шаг 5: Смоук-тест — поток покупки

От имени тестового пользователя Telegram (не админа):

1. **Откройте бота**, отправьте `/start`
   - [ ] Бот отвечает приветственным сообщением и инлайн-клавиатурой

2. **Нажмите «Buy access»** или отправьте `/buy`
   - [ ] Бот присылает счёт с депозитным адресом TRC20

3. **Отправьте USDT** (точную сумму счёта) с кошелька тестнета Nile на депозитный адрес
   - [ ] Запишите хеш транзакции: `____________________`

4. **Подождите** ~1–2 минуты, пока сработает cron `scan-payments`
   - [ ] Запустите вручную: `curl -H "Authorization: Bearer $CRON_SECRET" http://localhost:3000/api/cron/scan-payments`
   - [ ] Проверьте таблицу `invoices`: статус должен быть `paid`, `paid_tx_hash` заполнен

5. **Проверьте подписку**
   - [ ] В таблице `subscriptions` есть строка со статусом `active`
   - [ ] `/start` пользователя теперь показывает «Your access is active until …»

6. **Проверьте доступ к каналу**
   - [ ] Пользователь получил ссылку-приглашение или был добавлен в канал
   - [ ] Если это приглашение в канал, пользователь может успешно вступить

### Шаг 6: Смоук-тест — реферальный поток

1. **Получите реферальный код L1** от первого пользователя (сообщение `/start` или `ref_code` в БД)

2. **Второй пользователь Telegram** отправляет `/start <L1_REF_CODE>`
   - [ ] `users.parent_ref_code` устанавливается в код L1

3. **Второй пользователь покупает** (повторите шаги 2–6 из Шага 5)
   - [ ] У пользователя L1 появляются строки комиссии в `commission_ledger` (level=1)
   - [ ] `/start` пользователя L1 показывает реферальную активность

4. **Третий пользователь Telegram** отправляет `/start <L2_REF_CODE>` (используйте код второго пользователя)
   - [ ] Третий пользователь покупает
   - [ ] Пользователь L2 (второй) получает комиссию L1
   - [ ] Пользователь L1 (первый) получает комиссию L2 (level=2)

### Шаг 7: Смоук-тест — продление

1. **Первый пользователь** (с активной подпиской) отправляет `/renew`
   - [ ] Бот предлагает варианты тарифов для продления
   - [ ] Выберите тариф, оплатите сумму счёта
   - [ ] Новая подписка начинается с `ends_at` старой подписки (стекинг)
   - [ ] Время не теряется

2. **Если подписка истекла**, продлите и убедитесь, что `starts_at` = сейчас (без стекинга)

### Шаг 8: Смоук-тест — истечение + напоминания

1. **Временно установите `ends_at` подписки** на 1 час вперёд:
   ```sql
   UPDATE subscriptions SET ends_at = now() + interval '1 hour' WHERE status = 'active' LIMIT 1;
   ```

2. **Запустите expire-access:**
   ```bash
   curl -H "Authorization: Bearer $CRON_SECRET" http://localhost:3000/api/cron/expire-access
   ```

3. **Проверьте:**
   - [ ] Напоминание в личку отправлено (окно T-1h)
   - [ ] Строка `nudges_sent` существует с window='T-1h'
   - [ ] Нет преждевременного удаления (ends_at всё ещё > сейчас)

4. **Установите ends_at на сейчас:**
   ```sql
   UPDATE subscriptions SET ends_at = now() WHERE id = '<SUB_ID>';
   ```

5. **Снова запустите expire-access. Проверьте:**
   - [ ] Выполнено мягкое удаление (бан на 35с + разбан)
   - [ ] Статус подписки = `expired`
   - [ ] Напоминание повторно не отправлено (идемпотентность)

### Шаг 9: Смоук-тест — выплата

1. **Пользователь L1 задаёт адрес выплаты:**
   - Отправьте «Set payout address» (кнопка в меню бота)
   - Ответьте адресом TRC20 в Nile

2. **Проверьте, что комиссии готовы к выплате:**
   ```sql
   SELECT * FROM commission_ledger WHERE beneficiary_id = '<USER_ID>' AND status = 'payable';
   ```
   Если какие-то имеют статус `accrued` и прошли `unlock_at`, cron выплат переведёт их в `payable`.

3. **Запустите выплату:**
   ```bash
   curl -H "Authorization: Bearer $CRON_SECRET" http://localhost:3000/api/cron/payout-queue
   ```

4. **Проверьте:**
   - [ ] Создана строка `payout_batches` со `status = 'broadcast'`
   - [ ] `tx_hash` установлен в пакете
   - [ ] Строки `commission_ledger` помечены как `paid`
   - [ ] В обозревателе Nile USDT поступил на адрес пользователя L1
   - [ ] Запишите хеш транзакции выплаты: `____________________`

### Шаг 10: Смоук-тест — смётывание (sweep)

1. **Проверьте, что на депозитном адресе есть USDT:**
   Используйте TronGrid или TronService бота, чтобы проверить адрес из Шага 5.

2. **Запустите sweep:**
   ```bash
   curl -H "Authorization: Bearer $CRON_SECRET" http://localhost:3000/api/cron/sweep
   ```

3. **Проверьте:**
   - [ ] `invoices.swept` = `true` для оплаченного счёта
   - [ ] `invoices.sweep_tx_hash` установлен
   - [ ] USDT поступил на холодный кошелёк в обозревателе Nile
   - [ ] Запишите хеш транзакции смётывания: `____________________`

### Шаг 11: Админ + безопасность

1. **Админ-статистика:**
   - [ ] Админ отправляет `/admin` → показана статистика (оплаченные счета, баланс горячего кошелька, состояние kill switch)
   - [ ] Не-админ отправляет `/admin` → нет ответа или «not authorized»

2. **Kill switch — покупки:**
   ```bash
   curl -X POST -H "Authorization: Bearer $ADMIN_API_SECRET" \
     -H "Content-Type: application/json" \
     -d '{"buyDisabled":true,"reason":"smoke test"}' \
     http://localhost:3000/api/admin/kill-switch
   ```
   - [ ] Пользователь отправляет `/buy` → бот говорит, что покупки отключены
   - [ ] Включите обратно: тот же curl с `"buyDisabled":false`

3. **Kill switch — выплаты:**
   - [ ] Отключите выплаты, запустите cron выплат, проверьте EMPTY_RESULT
   - [ ] Включите выплаты обратно

4. **Эндпоинт здоровья:**
   ```bash
   curl http://localhost:3000/api/health
   ```
   - [ ] Возвращает 200 с числом ожидающих счетов и состоянием kill switch

## Чек-лист продакшен-выката

### Настройка кошельков

- [ ] **Холодный кошелёк** создан офлайн (изолированная от сети машина или аппаратный кошелёк). Адрес записан. Приватный ключ НИКОГДА не попадает на машину с доступом в интернет.
- [ ] Сгенерирован приватный ключ **горячего кошелька**. Пополнен:
  - [ ] ≥ 2 дней ожидаемого объёма комиссий в USDT
  - [ ] ≥ 100 TRX на пополнения газа для смётывания
- [ ] **Депозитный xprv** сгенерирован офлайн. В `TRON_DEPOSIT_XPRV` хранится только зашифрованная форма.
- [ ] `TRON_COLD_WALLET_ADDRESS` соответствует офлайн холодному кошельку.
- [ ] `TRON_HOT_WALLET_PK` соответствует горячему кошельку.

### Настройка канала

- [ ] Бот назначен админом в целевом приватном канале
- [ ] У бота есть право `invite users` (для выдачи подписок)
- [ ] У бота есть право `ban users` (для мягких удалений при истечении)
- [ ] Переменная окружения `DEFAULT_CHANNEL_ID` соответствует каналу

### Telegram-бот

- [ ] Токен бота от @BotFather (продакшен-бот, не тестовый)
- [ ] Webhook зарегистрирован на продакшен-`DEPLOY_URL`
- [ ] `TELEGRAM_WEBHOOK_SECRET` задан (случайный ≥16 символов)
- [ ] Команды бота настроены в @BotFather:
  - `start - Start the bot`
  - `buy - Buy channel access`
  - `renew - Renew your subscription`
  - `withdraw - Withdraw commissions`

### База данных

- [ ] Поднята продакшен-БД Neon
- [ ] `DATABASE_URL` задан (строка подключения с SSL)
- [ ] Применены миграции: `npx tsx scripts/migrate.mts`
- [ ] Применён сид: `npx tsx scripts/seed.ts`
- [ ] Проверены `subscription_plans` — цены и длительности корректны
- [ ] Проверен `commission_config` — пороги тарифов, ставки, минимальная выплата корректны
- [ ] `ops_kill_switch` = `(false, false)` — нет случайных блокировок

### Развёртывание на Vercel

- [ ] Все переменные окружения заданы в панели Vercel (включая `CRON_SECRET`, `ADMIN_API_SECRET`)
- [ ] Cron-задачи Vercel включены и указывают на продакшен-URL:
  - `scan-payments`: `POST /api/cron/scan-payments` каждую 1 минуту
  - `expire-access`: `POST /api/cron/expire-access` каждую 1 минуту
  - `payout-queue`: `POST /api/cron/payout-queue` каждые 5 минут
  - `sweep`: `POST /api/cron/sweep` каждые 10 минут
- [ ] У cron-задач настроен заголовок `Authorization: Bearer <CRON_SECRET>`
- [ ] Сборка проходит на продакшене
- [ ] Нет незакоммиченных локальных изменений

### Наблюдаемость

- [ ] Настроен Sentry DSN (`SENTRY_DSN` или `next.config.ts` с конфигом `sentry`)
- [ ] Sentry принимает события (вызовите тестовую ошибку)
- [ ] Настроен Logtail или эквивалентный сток логов
- [ ] `/api/health` возвращает 200 с корректными данными
- [ ] Внешний мониторинг аптайма (Better Uptime / Checkly) опрашивает `/api/health`
- [ ] Настроено оповещение при ответе `/api/health` не 200

### Лимиты безопасности

- [ ] `MAX_PAYOUT_PER_TX_USDT` задан разумным пределом (потранзакционный предохранитель)
- [ ] `MAX_PAYOUTS_PER_HOUR` задан разумным лимитом частоты
- [ ] `ADMIN_TG_IDS` задан (числовые ID пользователей Telegram через запятую)
- [ ] `ADMIN_API_SECRET` задан (случайный ≥16 символов, отличный от `CRON_SECRET`)
- [ ] Админ может получить доступ к `/admin` и видит статистику

### Перед запуском

- [ ] Смоук-тест в тестнете Nile завершён (все шаги из раздела выше пройдены)
- [ ] Продакшен горячий кошелёк пополнен
- [ ] Купите подписку сами как первый реальный пользователь — проверьте весь поток
- [ ] Друг/коллега покупает как второй пользователь по вашему реферальному коду — проверьте комиссию
- [ ] Мониторьте логи в первые 24 часа на предмет аномалий
- [ ] Настройте ежедневную сверку комиссий (сопоставляйте суммы `commission_ledger.paid` с оттоками горячего кошелька)
